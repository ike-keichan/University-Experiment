　語句の説明


  • ・アクセシビリティ
  • 特にWebサイト上における、情報やサービスへのアクセスのしやすさのことである。
    あらゆる人々がどのような環境下においてもスムーズにWebサイトを利用しやすいものであることである。
    ここでの「あらゆる人々」とは高齢者や障がい者などの身体的制約を持った全ての人々のことを指し、
    また、「どのような環境」とはWebサイトを閲覧するための機器の性能や閲覧する際の周囲の状況（うるさい場所、暗い場所など）のことを指す。

  • ・バリデーション
  • 一般的には検証、実証、認可、妥当性確認などの意味を持つ英単語。
    IT分野では、対象が適切に記述・構築・設計などがされているかどうかを検証することである。

  • ・それぞれの関係性
  • W3Cはいくつかのガイドラインを作っており、W3Cの規格に合うWebサイトはアクセシビリティとバリデーションの両方の面で一定以上の水準を満たしていることを意味する。
    HTMLやCSSはその性質上、いい加減な記述でも実行できてしまう。そのため、HTMLやCSSの記述量が多くなる複雑なWebサイトほどW3Cの規格に合うのは難しくなる。

ツールの説明


  • ・The W3C Markup Validation Service
  • W3Cによって作られた、HTMLで書かれた文書が適切に書かれているかどうかを検証するためのサービスである。

  • ・Web Content Accessibility Guidelines
  • W3CのWAIによって作られた、Webページがあらゆる人々にとって使いやすいことを示したウェブアクセシビリティに関するガイドラインである。

  • ・W3C CSS 検証サービス
  • W3Cによって作られた、HTMLで書かれた文書に対してCSSが適切に書かれているかどうかを検証するためのサービスである。
